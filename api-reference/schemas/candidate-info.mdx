---
title: 'CandidateInfo'
description: 'Candidate information schema for identity matching and compliance evaluation'
---

The CandidateInfo object contains personal information about the candidate being evaluated. This information is used for identity matching against criminal records and for compliance rule evaluation.

## Optional Fields

All fields are optional, but providing more information improves accuracy of identity matching and compliance evaluation.

<ParamField name="first_name" type="string">
  Candidate's first name **Example:** `"John"`
</ParamField>

<ParamField name="middle_name" type="string">
  Candidate's middle name or initial **Example:** `"A"` or `"Michael"`
</ParamField>

<ParamField name="last_name" type="string">
  Candidate's last name **Example:** `"Doe"`
</ParamField>

<ParamField name="date_of_birth" type="string">
  Date of birth in YYYY-MM-DD format. Used for identity matching and age-based compliance rules. <br />
  **Example:** `"1990-05-15"`
</ParamField>

<ParamField name="ssn" type="string">
  Social Security Number in XXX-XX-XXXX format. Used for identity matching. <br />
  **Example:** `"123-45-6789"`
</ParamField>

<ParamField name="address" type="string">
  Full address including street, city, state, and zip code. Used for address-based matching and compliance rules. <br />
  **Example:** `"123 Main St, Los Angeles, CA 90001"`
</ParamField>

<ParamField name="annual_salary" type="number">
  Annual salary in USD. Used for salary-based compliance rules and position level evaluation. <br />
  **Example:** `75000.0`
</ParamField>

<ParamField name="source_type" type="string">
  Data source identifier indicating where the criminal record data originated. <br />
  **Example:** `"court_records"`, `"state_repository"`, `"federal_database"`
</ParamField>

<ParamField name="source_reliability_rating" type="string">
  Rating of source reliability indicating the trustworthiness of the data source. <br />
  **Example:** `"high"`, `"medium"`, `"low"`
</ParamField>

<ParamField name="verification_status" type="string">
  Verification state indicating whether the candidate information has been verified. <br />
  **Example:** `"verified"`, `"pending"`, `"unverified"`
</ParamField>

<ParamField name="veteran_status" type="string">
  Candidate's veteran status. <br />
  **Example:** `"veteran"`, `"non-veteran"`, `"unknown"`
</ParamField>

<ParamField name="industry_sector" type="string">
  Industry sector of the employer or job position. Used for industry-specific compliance rules. <br />
  **Example:** `"healthcare"`, `"financial_services"`, `"technology"`, `"transportation"`
</ParamField>

<ParamField name="job_duties" type="string">
  Description of job duties and responsibilities. Used for role-specific compliance evaluation. <br />
  **Example:** `"Patient care and medical procedures"`, `"Financial transactions and account management"`
</ParamField>

<ParamField name="employer_attestation" type="string">
  Employer attestation details or certification information. <br />
  **Example:** `"Employer certified candidate meets all requirements"`
</ParamField>

<ParamField name="registry_status" type="string">
  Registry status information. <br />
  **Example:** `"active"`, `"inactive"`, `"pending"`
</ParamField>

<ParamField name="registry_active" type="boolean">
  Whether the registry entry is currently active. <br />
  **Example:** `true`, `false`
</ParamField>

## Identity Matching

When candidate_info is provided, the API performs identity matching to ensure the criminal records match the candidate being evaluated. Matching is performed on:

- **Name**: First, middle, and last name fuzzy matching
- **Date of Birth**: Exact or near-match comparison
- **SSN**: Exact match when available
- **Address**: Address history matching

## Compliance Evaluation

Candidate information is used for compliance rule evaluation:

- **Age-based rules**: Date of birth determines age at time of offense
- **Salary-based rules**: Annual salary determines position level and applicable rules
- **State-specific rules**: Address and applicant_state determine state-specific compliance requirements
- **Industry-specific rules**: Industry sector determines applicable regulatory requirements
- **Role-specific rules**: Job duties determine position-specific compliance checks
- **Veteran status**: Used for veteran-specific compliance considerations

## Pass-Through Fields

The following fields are pass-through only and are stored in BigQuery for analytics and auditing purposes. They do not require processing or transformation:

- `source_type` - Data source identifier
- `source_reliability_rating` - Source reliability rating
- `verification_status` - Verification state
- `veteran_status` - Veteran status
- `industry_sector` - Industry sector
- `job_duties` - Job duties description
- `employer_attestation` - Employer attestation details
- `registry_status` - Registry status
- `registry_active` - Registry active status

## Validation Rules

- **Date Format**: date_of_birth must be in YYYY-MM-DD format
- **SSN Format**: If provided, ssn must follow XXX-XX-XXXX pattern
- **Salary**: annual_salary must be a positive number if provided
