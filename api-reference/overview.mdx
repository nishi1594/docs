# Public Records API

The Public Records API provides comprehensive criminal record evaluation for background screening, including identity matching, compliance assessment, and regulatory analysis.

## Base URL

<CodeGroup>```https://awp-ai-eval-framework-cr-dev-859798439223.us-south1.run.app ```</CodeGroup>

## Quick Start Example

Evaluate your first criminal record:

```bash
curl -X POST "https://awp-ai-eval-framework-cr-dev-859798439223.us-south1.run.app/evaluate" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "record": {
      "xml": "<ScreeningResults>...</ScreeningResults>",
      "search_id": "76190bc0-e58c-494f-92c5-0c9a47389528",
      "search_date": "2024-01-15",
      "applicant_state": "CA",
      "candidate_info": {
        "first_name": "John",
        "last_name": "Doe",
        "date_of_birth": "1990-05-15",
        "source_type": "court_records",
        "verification_status": "verified"
      }
    }
  }'
```

## Core Endpoints

| Category      | Description                               | Example Endpoint                                  |
| ------------- | ----------------------------------------- | ------------------------------------------------- |
| Health        | Service health and status checks          | `GET /health`                                     |
| Authentication| Exchange API key for JWT token            | `POST /auth/token`                                |
| Evaluate      | Evaluate a single criminal record         | `POST /evaluate`                                  |
| Batch Evaluate| Process multiple records in batch         | `POST /evaluate-batch`                            |
| Identity Match| Compare candidate identity against record subject | `POST /complianceEngine/identity-match`           |

## Key Features

The Public Records API provides comprehensive background check capabilities:

- **Identity Matching** - Advanced fuzzy matching algorithms to verify candidate identity against criminal records
- **Compliance Evaluation** - Automated evaluation against FCRA and state-specific regulations
- **Multi-Engine Analysis** - Combines rule-based and AI-powered evaluation for comprehensive assessments
- **Regulatory Compliance** - Built-in support for federal and state reporting restrictions, clean slate laws, and industry-specific requirements

## Authentication

All endpoints (except `/health` and `/auth/token`) require JWT Bearer token authentication:

```http
Authorization: Bearer YOUR_JWT_TOKEN
```

Learn more about [authentication](/authentication).


## Getting Started

### 1. Authentication

Obtain your API key from your account manager and exchange it for a JWT token:

```bash
POST /auth/token
```

See [Authentication](/authentication) for full documentation.

### 2. Evaluate a Record

Submit a criminal record for evaluation:

```bash
POST /evaluate
```

See [Evaluate Endpoint](/api-reference/endpoints/evaluate) for full documentation.

### 3. Identity Matching

Compare candidate information against criminal record subjects:

```bash
POST /complianceEngine/identity-match
```

See [Identity Match Endpoint](/api-reference/endpoints/identity-match) for details.

### 4. Batch Processing

Process multiple records efficiently:

```bash
POST /evaluate-batch
```

See [Batch Evaluate Endpoint](/api-reference/endpoints/evaluate-batch) for details.

## Error Handling

The API uses standard HTTP status codes:

- `200` - Success
- `206` - Partial success
- `400` - Bad Request
- `401` - Unauthorized
- `422` - Validation Error
- `500` - Server Error
